SVT Database Browser

Browse SVT hardware configuration as stored in the Database. By default the window displays the active configuration. The 'View' panel displays
  • the mapset/hwset filenames with their respective CRCs and timestamps.
  • the names of the Pattern, SSMap, fcon files for each of the 12 wedges. For fcon, only one filename (barrel 0) for each wedge is displayed.
One can display SVT configuration for an arbitray Run Number for which the configuration exists. The 'Reload Active' once again restores the active configuration.

The 'Operation' panel allows one to
  • display one file at a time
  • save selected file (s)
  • save all the files shown in the list in the current directory
  • validate or compute CRCs of the selected file(s)
A Pupup Menu has been attached to the list in the 'Operations' panel to
  • select Multiple element selection mode
  • clear existing selection
  • select all the items
  • display names of all the mapset files present in DB
  • display names of all the hwset file present in DB
  • return to the active configuration
I've used Bill's C code from Java for validation or computation of CRC.
