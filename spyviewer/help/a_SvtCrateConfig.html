Spy Buffer Analysis Configuration

Configures the spymon analysis options in the SVT crates. A single message is created, either for one crate at a time or a common one valid for all the crates and published via SmartSockets. The configuration options are described in detail below.
  • Mode of monitoring,
    • wait for Run_Control partition to become available
    • do routine tasks even when the partition does not come into being.
  • Run Mode,
    • Routine pole and reading of spy buffers even if there is no freeze from outside. This option has to be elaborated in future to make it more flexible.
  • Spy buffer Analysis options
    • Analyse/Skip the buffer under consideration
    • Read Spy buffers
    • Dump to disk
    • Fill and publish histograms
    • Perform board/crate based simulation
  • For a selected buffer,
    • Frequency of buffer reading
    • Number of words to be read
    • Maximum number of words to be dumped
  • Type of histograms to be booked(?)/filled/published
    • Event Error histograms
    • Standard monitoring histograms
    • Histograms of physics quantities of interest
    • histograms to be consulted by SVT Expert
NB. As we do now we'll probably continue booking the histograms statically in the crate processes and only broadly modify the fill/publish options. If on the otherhand, we want to configure options on a histogram-by-histogram basis, it becomes much more involved. In that case the extra step will be to read somehow the list of booked histograms from each crate and then select them for filling/publishing.
